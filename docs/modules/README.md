# 模块化开发记录

本目录按六边形架构和实施计划拆分后续开发模块。每个模块必须满足以下要求后才标记为完成：

1. 核心接口位于 `domain/`、`ports/` 或稳定的应用层，不把外部系统类型带入核心。
2. 先编写并运行失败测试，再实现最小代码，最后完成全量回归。
3. 模块完成后新建独立目录，并在目录内生成中文 `README.md`，记录范围、架构边界、文件、测试、风险和 Git 提交。
4. 模块之间只通过端口、DTO、事件或明确的应用服务交互，禁止跨模块访问内部存储结构。
5. 提交信息使用中文，代码中的注释和文档说明使用中文；类名、函数名、命令和第三方产品名保留标准英文。

## 完成记录规范

模块完成后，在 `docs/modules/<编号>-<英文短名>/README.md` 中记录：

- 模块目标和非目标；
- 端口、适配器和依赖方向；
- 新增或修改的关键文件；
- 测试命令及实际结果；
- 安全、兼容性和后续扩展注意事项；
- 中文 Git commit 哈希和提交信息。
